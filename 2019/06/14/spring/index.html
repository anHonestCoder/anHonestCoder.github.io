BeanDefinition包含哪些bean metadata

  • A package-qualified class name: typically, the actual implementation class of the bean being defined.
  • Bean behavioral configuration elements, which state how the bean should behave in the container (scope, lifecycle callbacks, and so forth).
  • References to other beans that are needed for the bean to do its work. These references are also called collaborators or dependencies.
  • Other configuration settings to set in the newly created object — for example, the size limit of the pool or the number of connections to use in a bean that manages a connection pool.

Bean的注入方式

  • XML配置文件<bean>标签注入
  • @Component @Service等注解注入方式
  • @Bean @Config基于java代码的注入方式
  • 通过ApplicationContext注入外部已经存在的bean对象：调用ApplicationContext.getBeanFactory()方法会返回DefaultListableBeanFactory,可以通过DefaultListableBeanFactory 的registerSingleton(..)和registerBeanDefinition(..)方法注入外部bean。

Bean生命周期内的回调方式

  • InitializingBean (@PostConstruct)
  • BeanPostProcessor
  • DisposableBean (@PreDestroy)

Naming Beans

每个bean在spring container内都有唯一标识符（identifier），通常情况下一个bean只有一个唯一的标识符，不过spring支持为bean定义别名（aliases）。

当通过XML方式注入bean时，可以通过id和name属性指定bean identifier。也可以不设置id和name属性，如果没有设置这两个属性的话，容器会为bean生成一个唯一的name。

注解方式注入bean时如果没有指定name属性，生成name的规则是:With component scanning in the classpath, Spring generates bean names for unnamed components, following the rules described earlier: essentially, taking the simple class name and turning its initial character to lower-case.

Instantiating Beans

BeanDefinition的Class属性的在实例化对象时的两种用法：

  • 通过构造函数和反射实例化对象

  • 通过静态工厂方法实例化对象

  • 通过实例工厂方法实例化对象

依赖

依赖注入（Dependency Injection）

依赖注入是指在对象定义时通过构造方参数，工厂方法参数或者属性设置他们的依赖对象，但是容器会在对象创建完之后再去注入这些依赖对象。

依赖注入主要有两种方式：

  • 基于构造函数的依赖注入
  • 基于Setter方法的依赖注入

    基于构造函数的依赖注入

    基于构造函数的依赖注入参数匹配方式：

  • 默认按顺序匹配

  • 按参数类型匹配（type属性）

  • 指定参数顺序（index属性）

  • 根据参数名称匹配（name属性）

基于Setter方法的依赖注入

Setter-based DI is accomplished by the container calling setter methods on your beans after invoking a no-argument constructor or a no-argument static factory method to instantiate your bean.
